# quarto-revealjs-freestyle

<img width="400" height="400" alt="freestyle-icon" src="https://github.com/user-attachments/assets/5dca92c4-35ae-4882-a8a6-b280adb0ae38" />


An advanced fork of Emil Hvitfeldt's [quarto-revealjs-editable](https://github.com/EmilHvitfeldt/quarto-revealjs-editable)

**Freestyle** lets you drag, resize and reposition (almost) anything on your Reveal.js slides — text divs, images, equations, blockquotes, and even interactive code-cells and Plotly figures.

## What's new compared to editable

- **Robust initialization** — works reliably on browser reload (fixes blank elements on refresh)
- **Animated glow indicator** — editable elements pulse softly at rest so you always know what's draggable; glow intensifies on hover
- **Floating toolbar** — a non-intrusive bar slides in from the bottom whenever you make a change, with **Copy to clipboard** and **Save edits** buttons and visual feedback
- **Better initial sizing** — text divs start at 80% slide width with `height: auto`; images respect their natural aspect ratio
- **Aim at having a better correct initial positioning** — still not there - feel free to help and make a pull request if you find a solution for the elements to be anchored at their natural rendered position
- **UTF-8 support** — accented characters and special symbols are preserved when saving
- **Shortcode support** — `{{< meta ... >}}` and other Quarto shortcodes are now preserved correctly on save (fixes [editable#15](https://github.com/EmilHvitfeldt/quarto-revealjs-editable/issues/15))
- **Backslash support** — backslashes in equations and code are no longer corrupted on save (fixes [editable#16](https://github.com/EmilHvitfeldt/quarto-revealjs-editable/issues/16))
- ... 

## Installing

```bash
quarto add zinc75/quarto-revealjs-freestyle
```

This will install the extension under the `_extensions` subdirectory.
If you're using version control, you will want to check in this directory.

## Using

Declare the extension in your YAML front matter:

```yaml
revealjs-plugins:
  - editable
filters:
  - editable
```

It's also better to keep 

```yaml
    code-copy: false
```
to avoid the duplication of the *freeformed*  code cell after validation / export of modifications.

Mark any element as draggable with the `.freestyle` class:

```markdown
![](image.png){.freestyle}
```

```markdown
::: {.freestyle}
Some text, an equation, a blockquote, a callout...
:::
```

Once you have repositioned and resized your elements, use the floating toolbar at the bottom of the slide to **Copy to clipboard** or **Save edits**. Paste the clipboard content into your `.qmd` (or replace it with the saved file) and re-render — your elements will be locked in place and `freestyle`attributes will be replaced by [absolute positions](https://quarto.org/docs/presentations/revealjs/advanced.html#absolute-position) and new text attributes if needed.

> [!NOTE]
> Holding **Shift** while resizing an image preserves its aspect ratio.

> [!NOTE]
> The filter injects the `.qmd` source into the rendered HTML (base64-encoded) as long as there is at least one `{.freestyle}`  element in the source `.qmd` file, so that the save/copy feature can rewrite absolute positions. If you don't copy / paste or replace the `.qmd` generated by freestyle after inspection and modification, the html file will keep content encoded as non human-readable base64. It's OK if you want to obfuscate your content, but in general you should either remove `{.freestyle}` from the `.qmd` document for distribution (which is automatically done after having copied to clipboard or saved from the bottom floating toolbar.

> [!WARNING]
> {.freestyle} Images with a `width` (e.g. `{.editable width=10%}`) are **not** supported — leave width since freestyle will help you to position and size your image.

> [!WARNING]
> Images with a `<figcaption>` (Quarto markdown caption syntax) are not fully supported — the caption does not follow the image when dragged. Use images without captions for best results.

> [!TIP]
> If images don't stay the size you dragged them to, add the following to your SCSS theme:
> ```scss
> .reveal img {
>   max-width: unset;
>   max-height: unset;
> }
> ```

## Contributing

Contributions are very welcome! I maintain this extension alongside an intense professional activity, so I may not always be able to respond quickly or implement new features on demand. Well-documented pull requests are the best way to get things done.

If you find a bug or want to add a feature, feel free to open an issue or submit a pull request. A few known areas where help would be appreciated:

- **Initial positioning** — elements don't always anchor at their natural rendered position; a robust fix would be great
- **Callout blocks** - they are well target for moving / resizing, but they are somehow duplicated when rendering back the `.qmd` file. This is not the case with blockquotes, and I still don't figure why (any Pandoc expert ?)
- **Figure captions** — making the `<figcaption>` follow the image when dragged
- **Percentage widths** — supporting `width=X%` on images relative to the slide

If you submit a pull request, please keep the code style consistent and document your changes clearly in the commit message.

Happy freestyling !
